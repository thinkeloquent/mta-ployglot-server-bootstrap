# .Agent.md — `fastify_server` (Node / TypeScript / Fastify 5)

> LLM contract for agents working inside this package. Read before editing code, adding addons, or consuming the library. Mirrors `fastapi_server/.Agent.md` — behavior must stay parity-equivalent.

---

## 1. What this package is

Runtime-neutral **bootstrap orchestrator** for Fastify. It discovers user files at boot (`*.env.mjs`, `*.lifecycle.mjs`, `*.routes.mjs`), runs them through a priority-ordered **addon pipeline**, and returns a ready-to-listen `FastifyInstance`. It does **not** ship business code.

- Package name on npm: `@thinkeloquent/fastify-server`
- Type: `module` (ESM only). Compiled output at `dist/`.
- Node: `>=20`
- Parity counterpart: `../fastapi_server` (Python/FastAPI) — the two are lockstep.

## 2. Contracts (SOURCE OF TRUTH)

The JSON Schemas under `contracts/` are **normative**. All code, examples, and docs must conform.

| Schema | $id | Governs |
| --- | --- | --- |
| `contracts/bootstrap.schema.json` | `https://polyglot.local/bootstrap.schema.json` | Shape of the user-supplied `BootstrapConfig` object |
| `contracts/hooks.schema.json`     | `https://polyglot.local/hooks.schema.json`     | `onInit` / `onStartup` / `onShutdown` signatures |
| `contracts/loader.schema.json`    | `https://polyglot.local/loader.schema.json`    | `LoaderReport` emitted by every addon |

Parity rule: these three files are **byte-for-byte identical** with `../fastapi_server/contracts/`. Enforced by `../fastapi_server/tests/test_contract_parity.py`. When you change one, copy to the peer and re-run parity tests in the same PR.

### BootstrapConfig (abridged — see `src/contract/types.ts`)

```ts
interface BootstrapConfig {
  title?: string;
  port?: number;                   // 0..65535
  host?: string;                   // default "0.0.0.0"
  profile?: string;
  logger?: { level?: "trace"|"debug"|"info"|"warn"|"error"|"fatal"|"silent"; json_format?: boolean };
  paths?: {
    environment?: string | string[];
    lifecycles?:  string | string[];
    routes?:      string | string[];
    apps?:        string | string[];
  };
  initial_state?: Record<string, unknown>;  // structuredClone'd onto each request
}
```

Paths are **relative to `SetupOptions.baseDir`** (defaults to `process.cwd()`). Schema names stay `snake_case` to match JSON — do not rename `json_format` or `initial_state`.

### LoaderReport (every addon returns one)

```ts
interface LoaderReport {
  name: string;
  discovered: number; validated: number; imported: number; registered: number; skipped: number;
  errors: { step: string; error: string; path?: string }[];
  details?: Record<string, unknown>;
}
```

Invariants:
- Non-negative counters.
- `errors[].step` is a short verb: `"discover" | "import" | "shape" | "register" | "run"`.
- An addon **must never throw** out of `run`; catch and push to `report.errors` via `reportError`. The registry falls back to an empty report with a single `run` error if an addon leaks an exception.

### Hook signatures

```ts
type HookFn<Server = unknown> = (server: Server, config: ResolvedBootstrapConfig)
  => void | Promise<void>;
```

All three (`onInit`, `onStartup`, `onShutdown`) accept `(server, config)` and may be sync or async. Return value is ignored. Hook failures are **logged, not thrown**.

## 3. Public API

Import from `@thinkeloquent/fastify-server`:

| Export | Purpose |
| --- | --- |
| `setup(adapter, addons, config, opts?)` | Builds and returns a ready `FastifyInstance`. Async. |
| `SetupOptions { baseDir?, defaults?, logger? }` | Override base dir, defaults, and logger. |
| `createFastifyAdapter(options?)` | Fastify 5 runtime adapter (options: `closeGraceMs`, `requestIdHeader`, `fastifyOptions`). |
| `environmentAddon` (priority 10) | Imports `*.env.mjs` / `*.env.js`. Side-effect only. |
| `lifecycleAddon` (priority 20) | Registers `onInit` / `onStartup` / `onShutdown` from `*.lifecycle.mjs`. |
| `routeAddon` (priority 30) | Mounts plugins from `*.routes.mjs`. |
| `Addon<Server>` | Interface for custom addons. |
| `Registry<Server>` | Container; supports priority ordering and failure isolation. |
| `composeAddons(...groups)` | Flatten groups into a single addon list. |
| `createLoaderReport(name)`, `reportError(report, step, err, path?)` | Report helpers. |
| `mergeConfig(defaults, user, baseDir) -> ResolvedBootstrapConfig` | Merge + resolve paths. |
| `resolvePaths(paths, baseDir) -> ResolvedPaths` | Resolve `string | string[]` entries into absolute lists. |
| `sortByNumericPrefix(paths)` | Deterministic ordering of discovered files. |
| `validateBootstrapConfig(obj)` | Throws on invalid config. |

### `setup` in one sentence

`setup` merges defaults+user config (→ `ResolvedBootstrapConfig`), creates the Fastify instance via the adapter, runs every addon in priority order (10 → 20 → 30 → custom), logs per-addon reports, `structuredClone`s `initial_state` onto every request, runs init hooks inline, schedules startup hooks on `onReady` and shutdown hooks on `onClose`, installs `close-with-grace`, and returns the instance.

## 4. Discovery rules — what the built-in addons look for

Each addon scans directories listed in `config.paths.<key>` (already resolved to absolute paths).

| Addon | Suffixes matched | Module contract |
| --- | --- | --- |
| `environmentAddon` | `.env.mjs`, `.env.js` | **Imported for side effects** (e.g. `process.env.X ??= "y"`). No exports required. |
| `lifecycleAddon`   | `.lifecycle.mjs`, `.lifecycle.js` | Named exports (or an object default): any of `onInit`, `onStartup`, `onShutdown`. Modules with none are counted as `skipped`. |
| `routeAddon`       | `.routes.mjs`, `.routes.js` | **Default export** (or `mount` named export) is a function `(fastify, config) => void \| Promise<void>`. Mounted via `server.register(async scope => fn(scope, config))` for isolation. |

Files are sorted by **numeric prefix** via `sortByNumericPrefix`. Use `01_`, `10_`, `20_` style filenames when ordering matters.

## 5. Usage examples

### 5.1 Minimal quickstart (see `examples/quickstart/`)

```js
import {
  setup,
  createFastifyAdapter,
  environmentAddon, lifecycleAddon, routeAddon,
} from "@thinkeloquent/fastify-server";

const server = await setup(
  createFastifyAdapter(),
  [environmentAddon, lifecycleAddon, routeAddon],
  {
    title: "my-service",
    port: 3000,
    paths: {
      environment: ["config/environment"],
      lifecycles:  ["config/lifecycles"],
      routes:      ["config/routes"],
    },
  },
);

await server.listen({ port: 3000, host: "0.0.0.0" });
```

### 5.2 User files

```js
// config/environment/01_defaults.env.mjs — side-effect only
process.env.BUILD_ID ??= "local-dev";

// config/lifecycles/10_startup.lifecycle.mjs
export function onInit(server, config) {
  server.log.info({ title: config.title }, "boot complete");
}
export async function onStartup(server) { /* warm caches */ }
export async function onShutdown(server) { /* drain */ }

// config/routes/10_healthz.routes.mjs
export default async function healthz(fastify, config) {
  fastify.get("/healthz", async () => ({ status: "ok", service: config.title }));
}
```

### 5.3 Custom addon (always return a report, never throw)

```ts
import type { Addon } from "@thinkeloquent/fastify-server";
import { createLoaderReport, reportError } from "@thinkeloquent/fastify-server";

export const metricsAddon: Addon = {
  name: "metrics",
  priority: 40,
  async run(server, _config, ctx) {
    const report = createLoaderReport("metrics");
    try {
      ctx.getAdapter().decorate(server, "_metrics", { counter: 0 });
      report.registered = 1;
    } catch (err) {
      reportError(report, "register", err);
    }
    return report;
  },
};
```

## 6. Invariants an agent must preserve

1. **Contract parity**: never edit `contracts/*.json` without mirroring to `../fastapi_server/contracts/` in the same PR.
2. **Addon isolation**: a throwing addon **must not** break the others. Catch inside `run`; push to `report.errors`.
3. **Report shape**: every addon returns a `LoaderReport` with all numeric counters ≥ 0 and an `errors` array (possibly empty).
4. **Priority semantics**: environment (10) runs before lifecycle (20) runs before routes (30). User addons should slot in using sparse integers (e.g. 15, 25) and document why.
5. **Deterministic ordering**: always sort discovered files with `sortByNumericPrefix` — tests depend on it.
6. **Path resolution is the orchestrator's job**: addons receive *absolute* paths in `config.paths.*`. Do not re-resolve.
7. **ESM only**: use `node:fs/promises`, `node:path`, `pathToFileURL(file).href` for dynamic `import()`. Don't regress to CommonJS.
8. **`.js` import specifiers for TS**: source files import peers as `"./foo.js"` even though the source is `foo.ts`. This is required by NodeNext. Preserve it.
9. **Encapsulation on routes**: always register user route plugins via `server.register(async scope => fn(scope, config))` — keeps a per-route plugin scope and request/response logging context.
10. **Schema key casing**: JSON schemas and `BootstrapConfig` keep `snake_case` (`json_format`, `initial_state`) even on the TS side — that's the cross-runtime contract.
11. **Mirror naming**: TS uses `camelCase` (`environmentAddon`, `onStartup`); Python uses `snake_case` (`environment_addon`, `on_startup`). Names differ — *behavior* doesn't.

## 7. Testing contract

- Tests live under `test/` and run via `node --test --import tsx test/*.test.ts` (see `package.json:scripts.test`).
- `test/contract.test.ts` covers `mergeConfig`, `resolvePaths`, `validateBootstrapConfig`, and report helpers.
- `test/registry.test.ts` covers priority ordering and failure isolation.
- Schema parity is enforced on the Python side (`../fastapi_server/tests/test_contract_parity.py`) — run it after any schema change.
- Build is pure TypeScript: `npm run build` → `dist/`. Publishing consumes `dist/` + `contracts/` only (see `package.json:files`).

When adding a new feature: (a) add a test in `test/`, (b) update this file's §3 or §4 if the public surface changed, (c) mirror to `fastapi_server` if it affects behavior or schemas.

## 8. Do / Don't for agents

**Do**
- Keep `src/index.ts` the single public import surface. Every export must be re-exported from a stable `exports` path in `package.json`.
- Use `createLoaderReport` + `reportError` — never build report objects by hand.
- Treat `ResolvedBootstrapConfig` as read-only after `mergeConfig` returns.
- Prefer `ctx.registerInitHook` / `registerStartupHook` / `registerShutdownHook` over poking Fastify hooks directly from an addon.
- Use `server.log` (pino) for structured logging; `ctx.logger` for bootstrap-layer logs.

**Don't**
- Don't publish from this repo directly — produce `Makefile.publish` targets and let the user trigger them. (See project memory.)
- Don't introduce cross-package imports from `fastapi_server/`.
- Don't rename public exports without updating `src/index.ts`, the `exports` map in `package.json`, the README, and the parity twin.
- Don't swallow errors silently — they must land in a `LoaderReport.errors` entry.
- Don't add runtime dependencies beyond `fastify` and `close-with-grace` unless discussed.
- Don't switch to CommonJS, drop `"type": "module"`, or change the `.js` import-specifier convention.
